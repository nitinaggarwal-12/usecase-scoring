# 🏛️ Enterprise Engineering Manifesto (Gemini Advanced Agentic Coding)

> **MANDATORY PERMANENT DIRECTIVE FOR ALL AGENT INVOCATIONS**
> Read and enforce this manifesto before generating or modifying any code in this repository.

---

## 1. Zero Static Mocks & Placeholders (Dynamic First)
* **Never ship static placeholders**: Writing static mock answers (e.g., hardcoding `88/100 Pts`) or repetitive dummy text is **UNACCEPTABLE**.
* **Real-Time Dynamic Evaluation**: If an intake box or prompt scanner is created, it **MUST** parse the input string dynamically, evaluate character specificity/keywords, and derive unique mathematical weights and economic calculations on the fly.

## 2. Enterprise-Grade Popup & Modal Restraint
* **No Opaque Oceans of White Space**: Never obscure the underlying dashboard with solid `#ffffff` or `rgba(255,255,255,0.95)` screens.
* **Premium Glassmorphic Overlays**: Popups must utilize highly premium dark glass backdrops (`rgba(15, 23, 42, 0.65)` in light mode, `rgba(0, 0, 0, 0.82)` in dark mode) with `blur(12px)`.
* **Proportional Restraint**: Standard dialog boxes must be tightly restrained (e.g., `max-width: 680px`) with high-elevation drop shadows (`0 25px 60px -15px rgba(0,0,0,0.5)`).

## 3. Strict State Lineage & URL Traceability
* **Explicit Unique IDs**: Every single assessment variation, clone, import, or pitch must generate a strict unique primary key (`id: 'assessment_' + Date.now()`).
* **Complete URL Sync**: Every interactive view must reflect its state explicitly in the URL query hash (e.g., `?id=UNIQUE_ID`) with real-time `hashchange` event listeners.

## 4. Elite Autonomous Verification (/goal)
* Act autonomously like an elite Principal Software Engineer. Validate edge cases, execute build verifications (`npm run build`), and verify zero-mock compliance before submitting any turn.
